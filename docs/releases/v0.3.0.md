# v0.3.0

This release adds infrastructure-focused catalog direction and a cleaner scanner-routing model.

## Added

- `INF` category direction
- `DOCKER`, `COMPOSE`, and `K8S` families
- starter Docker, Compose, and Kubernetes rules
- family-level `scanner_profile`
- optional rule-level `scanner_override`

## Improved

- scanner integration model for MSG
- generated documentation and browsing
- continued deterministic generation

## Why it matters

The catalog now better represents waste that appears in infrastructure and orchestration environments, not just in source code and application logic.

Because waste is a bug.
