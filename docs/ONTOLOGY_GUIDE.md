# Eco Rules Catalog — Ontology Guide

The Eco Rules Catalog uses an ontology to describe
how inefficiencies impact computing resources.

The ontology connects rules to sustainability outcomes.
````

---

## Ontology Purpose

The ontology allows rules to be analyzed across dimensions such as:

* resource consumption
* system layers
* detection strategies
* remediation patterns

This enables:

* sustainability reporting
* cross-language analysis
* system-wide optimization insights

---

## Resource Impacts

Resource impacts describe **what resources are wasted**.

Examples:

| Resource            | Meaning                           |
| ------------------- | --------------------------------- |
| CPU                 | unnecessary computation           |
| Memory              | excessive allocation or retention |
| Network             | excessive data transfer           |
| Storage             | unnecessary data persistence      |
| Energy              | indirect energy consumption       |
| Latency             | increased response time           |
| Infrastructure Cost | increased cloud cost              |

---

## Mechanisms

Mechanisms describe **why the inefficiency occurs**.

Examples:

| Mechanism             | Description                       |
| --------------------- | --------------------------------- |
| Redundant Computation | work repeated unnecessarily       |
| Excessive Allocation  | unnecessary memory allocation     |
| Network Amplification | excessive network requests        |
| Polling               | repeated checks instead of events |
| Cache Misuse          | ineffective caching               |
| Unbounded Growth      | resources growing indefinitely    |

---

## System Layers

System layers describe **where the inefficiency occurs**.

Examples:

| Layer          | Description              |
| -------------- | ------------------------ |
| Code           | application logic        |
| Runtime        | execution environment    |
| Data           | databases and storage    |
| Network        | communication protocols  |
| Architecture   | system design            |
| AI/ML          | machine learning systems |
| Organizational | governance and process   |

---

## Detection Methods

Detection methods describe **how inefficiencies are discovered**.

Examples:

* Static analysis
* Runtime profiling
* Query analysis
* Network tracing
* Log pattern detection
* CI pipeline inspection

---

## Remediation Patterns

Remediation patterns describe **typical solutions**.

Examples:

* algorithmic improvement
* caching
* batching
* event-driven design
* infrastructure right-sizing

---

## Why the Ontology Matters

Without an ontology, rules are isolated lint checks.

With the ontology, rules become part of a **systemic model of efficiency**.

This enables:

* sustainability metrics
* architectural insights
* automated reporting
* cross-platform analysis
